## Часть 1. Финансовый дашборд (репозиторий `finance`)

### 1.1. Верхние виджеты → «Главные цифры месяца» (высокий приоритет)

**Проблема.** Четыре верхних виджета (выручка / маржа / маржинальность / прибыль) дублируют «бизнес-уравнение» и **не совпадали по цифрам** (маржинальность сверху 34% vs 33% в уравнении). (05:26–07:50, 11:41–12:17)

**Что сделать (решение Кости):**

1. **⬜ Заголовок «Цели месяца» / «Главные цифры месяца».** Сейчас у `KpiGrid` заголовка нет ([kpi-grid.tsx:29](../src/components/dashboard/kpi-grid.tsx)). (27:30–28:05)
2. **⬜ Виджет всегда показывает активный (текущий) месяц**, без смены интервала. Сейчас наоборот: у KPI-блока свой селектор периода, по умолчанию = глобальный период (весь год, `startMonth=0…endMonth=11`) ([dashboard.tsx:64-97,245-263](../src/components/dashboard/dashboard.tsx)). (28:21–28:36)
3. **✅ Показатели как опережающие индикаторы vs план, с %.** Реализовано: карточка показывает «▲/▼ N% из {бюджет}» ([kpi-card.tsx:37-51](../src/components/dashboard/kpi-card.tsx), `deviation()` в [kpi-grid.tsx:16-19](../src/components/dashboard/kpi-grid.tsx)). Формат совпадает с «12 из 15, −19%». (29:36–30:35)
4. **🔶 Добавить «Запросы» как опережающий индикатор.** Показатель «Запросы» есть, но **в бизнес-уравнении**, не в верхних виджетах ([business-equation-chart.tsx:235,246](../src/components/dashboard/business-equation-chart.tsx)). Логика этапов **иная**, чем в ТЗ: у Бластера — лиды по дате «Бриф получен»; у Култа — «Первичный контакт» ([hint-texts.ts:60-66,200-206](../src/lib/hint-texts.ts)). Костя просил сумму 3 этапов amoCRM: «Подготовка КП» + «Ждём ответа на КП» + «Бриф передан в продакшен» — **нужно сверить, актуальна ли ещё эта формулировка** (логика запросов правилась после созвона, коммит `+ смена логики запросов`). (30:42–34:20)
5. **✅ «Расходы» — не добавлять.** В верхних виджетах расходов нет (`SHOW_EXTRA_KPIS=false`, [kpi-grid.tsx:14,73](../src/components/dashboard/kpi-grid.tsx)). (30:35–30:41)
6. **🔶 Соответствие с бизнес-уравнением + подсказки-формулы.** Подсказки — **✅ сделаны полностью** ([hint-texts.ts](../src/lib/hint-texts.ts) — «Источник/Факт/План» на каждый показатель, отдельно Култ/Бластер; режим «Подсказки» [hint-mode.tsx](../src/contexts/hint-mode.tsx)). Совпадение цифр: маржинальность Бластера уже считается как в KPI (Маржа/Выручка, [business-equation-chart.tsx:198-203](../src/components/dashboard/business-equation-chart.tsx)) — сближено, но **полное совпадение надо проверить визуально**. ### 1.3. Винрейт → Конверсия (высокий приоритет) — ✅ сделано аддитивно

> **Реализовано 06.07 (стратегия «только добавляем»):** к Бластеру добавлена «Конверсия» = **Победы ÷ Запросы** (план 30%) рядом с «Винрейтом»; Винрейт и «Победы» **не удалены**. Формула = «конверсия запросов в проекты» (проверено: факт ≈ 21%, −30% к плану). Файлы: `business-equation-chart.tsx`, `hint-texts.ts`. Итоговое «убрать винрейт» — по решению Кости позже.

**Исходное решение Кости (директивное):** убрать «Винрейт», оставить единую «Конверсию» как в Култе; если Лизе нужен винрейт — считает сама. (13:46–17:44)

**Состояние в коде — расходится с ТЗ:**
- Бластер **по-прежнему показывает «Винрейт»** (Победы ÷ Завершённые, план 30%) + отдельную колонку «Победы» ([business-equation-chart.tsx:248-249](../src/components/dashboard/business-equation-chart.tsx), [hint-texts.ts:67-82](../src/lib/hint-texts.ts)).
- Култ показывает «Конверсию» (Взятые в работу ÷ Запросы, план 16%) ([:236](../src/components/dashboard/business-equation-chart.tsx), [hint-texts.ts:207-213](../src/lib/hint-texts.ts)).
- То есть **два разных показателя, названия, определения и плана** — не унифицированы.
- **✅ Формула-подсказка** у обоих есть.

> ⚠️ **Требует решения.** Код активно правил винрейт ПОСЛЕ созвона (коммиты `30% винрейт`, `смена логики в винрейт`, `изменено победы`) — похоже, винрейт **оставили** (возможно, по просьбе Лизы), вопреки решению Кости «убрать». Нужно подтвердить у Кости: унифицировать в «Конверсию» или оставить как есть. Плюс определение «конверсии» в коде («Взятые в работу ÷ Запросы») **не совпадает** с тем, что Костя описал в созвоне («запросы → проекты / Реализовано»). (15:37–16:12)
